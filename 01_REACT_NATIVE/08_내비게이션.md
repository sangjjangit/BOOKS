# 08_내비게이션
- https://reactnavigation.org/

## 8.1 리액트 내비게이션
- 내비게이션 종류: 스택(stack), 탭(tab), 드로어(drawer) 내비게이션

- 내비게이션의 구조
- NavigationContainer 컴포넌트: 내비게이션의 계층 구조와 상태를 관리하는 컨테이너 역할, 최상위 컴포넌트
- Navigator 컴포넌트: 화면을 관리하는 중간 관리자 역할, 여러 개의 Screen 컴포넌트를 갖고 있다.
- Screen 컴포넌트: name(화면 이름), component(화면 컴포넌트) 속성 지정
- NavigationContainer(
  Navigator[
    Screen,
    Screen,
    ...
  ]
)
- 설정 우선 순위
- Navigator 컴포넌트 속성 지정하면 모든 자식 컴포넌트에 적용된다.
- 작은 범위의 설정일수록 우선순위가 높다.


## 8.2 스택 내비게이션
- npm install @react-navigation/stack
- 가장 많이 사용되는 내비게이션
- 현재 화면 위에 다른 화면을 쌓으면서 화면을 이동하는 것이 특징(push, pop)
- 첫 화면은 Navigator 컴포넌트의 첫 번째 자식 Screen 컴포넌트이다.
- Navigator 컴포넌트 initialRouteName 속성은 첫 번째 화면을 지정할 수 있다.

#### - 화면 이동: navigate 함수
#### - 헤더 수정
- 1. Screen 컴포넌트의 name 속성 변경
- 2. Screen 컴포넌트의 options 속성에서 headerTitle 값 변경

#### - 헤더 스타일 수정
- Navigator 컴포넌트의 screenOptions 속성의 headerStyle, headerTitleStyle 값 변경

#### - 헤더 타이틀 컴포넌트 변경
- Navigator 컴포넌트의 screenOptions 속성의 headerTitle 에 함수 지정

#### - 헤더 버튼 수정하기
- headerBackTitleVisible -> headerBackButtonDisplayMode

#### - 헤더 버튼 스타일 수정하기
- Screen 컴포넌트의 options 속성에서 headerTitleStyle, headerTintColor 값 변경

#### - 헤더 버튼 컴포넌트 변경
- Screen 컴포넌트의 options 속성에서 headerRight, headerLeft 값 변경
- useLayoutEffect Hook: 화면이 렌더링되지 전에 실행된다. useEffect Hook 과 사용법 동일.
- navigation.popToTop(): 모든 화면을 내보내고 첫 화면으로 돌아가는 기능

#### - 헤더 감추기
- Screen 컴포넌트의 options 속성에서 headerShown
- Navigator 컴포넌트의 screenOptions 속성에서 headerMode


## 8.3 탭 내비게이션
- 화면 위나 아래에 위차하며, 탭 버큰을 누르면 버튼과 연결된 화면으로 이동하는 방식으로 동작
- 채팅, 인스타그램, 유튜브 등 다수의 애플리케이션에서 사용
- npm install @react-navigation/bottom-tabs

#### - 화면 구성
- Navigator 컴포넌트 initialRouteName 속성은 첫 번째 화면을 지정.

#### - 탭 바 아이콘 설정
- Screen 컴포넌트의 options 속성에서 tabBarIcon 설정
- Navigator 컴포넌트의 screenOptions 속성을 사용해서 일괄 관리 가능

#### - 라벨 수정하기
- 라벨은 Screen 컴포넌트의 name 속성값이 기본값으로 사용, tabBarLabel 로 수정 가능
- 라벨의 위치 조정, tabBarLabelPosition
- 라벨이 안나오도록, tabBarShowLabel
- 스타일 지정, tabBarStyle
- 탭아이콘 선택시, 활성화/비활성화 색상,  tabBarActiveTintColor, tabBarInactiveTintColor
